## 一、项目愿景与目标

ProCyc Skill 商店是一个面向 3C 维修领域的开放技能生态平台，旨在：

- 为开发者提供标准化的技能封装、发布和交易环境。
- 为智能体市场提供丰富的可复用能力单元，降低智能体开发门槛。
- 与 FixCycle 主生态无缝集成，实现技能调用、计费、分成闭环。

## 二、总体架构

## 四、与现有项目的集成点

- **认证与用户**：复用 FixCycle 的 Supabase 认证体系，技能调用鉴权可使用 JWT。
- **计费**：接入 FCX 账户系统，技能调用扣除用户 FCX 余额。
- **智能体市场**：商店作为技能提供方，智能体市场作为消费方，两者通过 API网关交互。
- **数据源**：技能底层依赖 FixCycle 各模块的 API（如 M1、M4、Valuation），需确保 API 稳定并提供文档。

## 五、技术选型

| 组件       | 技术栈                          | 说明                       |
| ---------- | ------------------------------- | -------------------------- |
| 技能仓库   | GitHub                          | 代码托管，利用 GitHub 生态 |
| 商店前端   | Next.js + TailwindCSS           | 静态生成，部署在 Vercel    |
| 技能运行时 | Node.js / Python                | 支持两种主流语言           |
| 计费引擎   | Node.js + PostgreSQL            | 独立微服务                 |
| 鉴权       | JWT + 中间件                    | 与现有认证系统统一         |
| 搜索       | Lunr.js (前端) / Algolia (后期) | 轻量级搜索方案             |

## 六、风险与应对

| 风险                   | 应对                                 |
| ---------------------- | ------------------------------------ |
| 技能质量参差不齐       | 建立审核流程和认证计划               |
| 调用量激增导致计费不准 | 设计高可用计费系统，使用消息队列削峰 |
| 开发者贡献不活跃       | 定期举办活动，提供激励（FCX 奖励）   |
| 与主项目 API 依赖      | 保持 API 版本兼容，提供 mock 服务    |
