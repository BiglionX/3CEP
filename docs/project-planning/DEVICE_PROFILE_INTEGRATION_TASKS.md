# 设备档案集成原子任务清单

## 任务概述

基于现有LIFE设备生命周期管理模块，为M1、DIY、CROWDFUND、FCX模块补充设备档案集成功能。

## 原子任务详情

### M1模块任务

#### M1-105: 扫码落地页设备档案集成

- **描述**: 修改M1-103扫码落地页，增加档案展示区域并添加激活设备按钮
- **关联模块**: M1, LIFE
- **前置依赖**: LIFE-202, LIFE-201
- **技术要点**:
  - 调用LIFE-202获取设备档案信息
  - 实现设备档案标签页UI组件
  - 添加"激活设备"按钮调用LIFE-204
  - 展示设备基本信息、维修历史、保修状态

#### M1-106: AI诊断历史故障参考

- **描述**: 修改AI诊断逻辑，集成设备历史维修记录作为大模型上下文
- **关联模块**: M1, LIFE
- **前置依赖**: LIFE-202
- **技术要点**:
  - 诊断请求前调用LIFE-202获取历史记录
  - 构造包含历史故障的Prompt
  - 优化大模型推理参数
  - 验证诊断准确性提升效果

### DIY模块任务

#### DIY-204: 教程页面设备档案入口

- **描述**: 在DIY维修教程页面添加设备档案查看入口
- **关联模块**: DIY, LIFE
- **前置依赖**: LIFE-202
- **技术要点**:
  - 检测用户扫码来源的设备关联关系
  - 在教程页面合适位置添加"查看设备档案"链接
  - 实现设备相关教程的智能推荐算法
  - 优化用户学习路径体验

### CROWDFUND模块任务

#### CROWDFUND-304: 以旧换新设备档案评估

- **描述**: 在以旧换新流程中读取旧设备档案用于折价评估
- **关联模块**: CROWDFUND, LIFE
- **前置依赖**: LIFE-202
- **技术要点**:
  - 获取旧设备完整维修历史
  - 结合设备年龄、维修频率、故障类型计算折价系数
  - 实现动态报价算法
  - 提供透明的估价依据展示

### FCX模块任务

#### FCX-405: 工单完成自动记录维修事件

- **描述**: 工单服务完成后自动调用LIFE-201记录维修事件
- **关联模块**: FCX, LIFE
- **前置依赖**: LIFE-201
- **技术要点**:
  - 监听工单状态变更事件
  - 提取维修相关信息（时间、故障、配件等）
  - 调用LIFE-201写入设备档案
  - 更新设备生命周期状态

### n8n自动化任务

#### N8N-205: 工单完成自动化工作流

- **描述**: 创建监听工单表变化的工作流，触发档案记录和用户通知
- **关联模块**: n8n, LIFE, FCX
- **前置依赖**: n8n已集成, LIFE-201
- **技术要点**:
  - 配置Supabase Realtime监听器
  - 设计工作流触发条件和执行逻辑
  - 集成企业微信通知API
  - 实现异常处理和重试机制

#### N8N-206: 设备激活欢迎消息工作流

- **描述**: 监听设备激活事件，向用户发送个性化欢迎信息
- **关联模块**: n8n, LIFE
- **前置依赖**: n8n已集成
- **技术要点**:
  - 设置激活事件监听规则
  - 设计欢迎消息模板和个性化变量
  - 集成消息推送渠道（微信、短信等）
  - 实现用户引导流程

## 技术依赖关系图

```
LIFE模块API
├── LIFE-201 (记录设备事件)
├── LIFE-202 (查询设备档案)
└── LIFE-204 (激活设备)

业务模块调用关系
├── M1模块 → LIFE-202, LIFE-204
├── DIY模块 → LIFE-202
├── CROWDFUND模块 → LIFE-202
├── FCX模块 → LIFE-201
└── n8n工作流 → 各业务模块事件监听

数据流转
用户操作 → 业务模块 → LIFE API → 数据库 → 自动化通知
```
