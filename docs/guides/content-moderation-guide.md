# 内容审核系统使用指南

## 系统概述

内容审核系统提供完整的自动化和人工审核流程，确保平台内容质量和合规性。

## 核心功能

### 1. 自动审核机制

- 敏感词过滤(支持多语言)
- 图像内容识别
- 文本模式匹配
- 机器学习辅助检测
- 外部API集成

### 2. 人工审核工具

- 任务分配和跟踪
- 内容预览和对比
- 审核决策支持
- 质量管控机制
- 绩效统计分析

### 3. 违规处理流程

- 违规内容识别
- 处罚措施执行
- 申诉处理机制
- 记录和报告生成

## 使用说明

### 自动审核系统

访问路径: `/moderation/auto`

主要功能:

- 实时内容审核测试
- 审核结果展示
- 置信度评估
- 违规类型识别

### 人工审核工具

访问路径: `/moderation/manual`

操作流程:

1. 查看待审核任务列表
2. 选择任务查看详情
3. 进行内容审核判断
4. 添加审核备注
5. 提交审核决定

### 违规处理管理

访问路径: `/moderation/violations`

功能包括:

- 违规记录查询
- 处罚措施管理
- 申诉案件处理
- 统计报表生成

## 审核标准

### 内容禁止类型

- 违法违规内容
- 淫秽色情信息
- 低俗媚俗内容
- 暴力恐怖信息
- 虚假谣言信息
- 侵犯他人权益
- 其他违规内容

### 审核判定原则

- 合法合规性优先
- 社会公序良俗
- 平台社区规范
- 用户体验保护

## 最佳实践

1. **建立审核标准**: 明确各类内容的判定标准
2. **培训审核人员**: 定期进行审核技能培训
3. **质量控制**: 建立审核质量抽检机制
4. **申诉处理**: 建立公平透明的申诉流程
5. **数据统计**: 定期分析审核数据和趋势

## API接口

### 自动审核接口

```bash
POST /api/moderation/auto
{
  "content": "待审核的内容",
  "contentType": "text",
  "metadata": {
    "userId": "用户ID",
    "postId": "内容ID"
  }
}
```

### 人工审核接口

```bash
POST /api/moderation/manual/tasks
{
  "taskId": "任务ID",
  "decision": "approve|reject",
  "reason": "审核理由",
  "reviewer": "审核员ID"
}
```
