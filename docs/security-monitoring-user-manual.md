# 安全监控系统用户手册

## 📖 系统简介

安全监控系统是一个集成了实时威胁检测、异常行为分析和可视化监控的一体化安全平台。系统能够自动识别潜在的安全威胁，提供及时的告警通知，并通过直观的仪表板展示整体安全态势。

## 🚀 快速开始

### 访问系统

1. 打开浏览器访问: `http://localhost:3001/security-dashboard`
2. 系统将自动加载安全监控仪表板

### 基本操作

#### 查看安全概览

- 系统首页显示整体安全评分(0-100分)
- 绿色(≥90): 安全状况优秀
- 黄色(70-89): 安全状况良好
- 橙色(50-69): 需要关注
- 红色(<50): 存在严重风险

#### 监控实时威胁

- **威胁事件**: 显示检测到的威胁总数
- **异常行为**: 展示异常用户行为数量
- **高危用户**: 标识具有高风险评分的用户
- **攻击拦截**: 统计成功阻止的攻击次数

## 📊 主要功能模块

### 1. 安全仪表板

```
系统安全评分 | 合规状态 | 威胁指标概览
```

**功能说明**:

- 实时显示系统整体安全状况
- 展示GDPR、PCI DSS等合规状态
- 提供关键安全指标的即时数据

### 2. 威胁监控

```
近期告警 | 威胁来源排行 | 安全建议
```

**功能说明**:

- 按时间顺序展示最近的安全告警
- 统计威胁来源IP地址排名
- 提供针对性的安全改进建议

### 3. 用户风险管理

```
风险用户列表 | 行为分析 | 风险趋势
```

**功能说明**:

- 识别和标记高风险用户
- 分析用户行为模式异常
- 展示风险评分变化趋势

## 🔧 系统管理

### API接口使用

#### 1. 获取系统状态

```bash
GET /api/security-monitoring?action=status
```

响应示例:

```json
{
  "success": true,
  "data": {
    "enabled": true,
    "detectionEnabled": true,
    "anomalyDetectionEnabled": true,
    "alertingEnabled": true,
    "activeAlertCount": 0
  },
  "timestamp": "2026-02-27T13:52:03.498Z"
}
```

#### 2. 处理安全事件

```bash
POST /api/security-monitoring
Content-Type: application/json

{
  "action": "process_event",
  "eventData": {
    "eventType": "failed_login",
    "userId": "user_123",
    "ipAddress": "192.168.1.100",
    "userAgent": "Mozilla/5.0...",
    "details": {
      "attemptCount": 5,
      "timeWindow": 300
    }
  }
}
```

#### 3. 获取安全仪表板数据

```bash
GET /api/security-monitoring?action=dashboard
```

### 配置管理

## ⚠️ 告警级别说明

### 威胁等级分类

- **LOW (低)**: 潜在风险，建议关注
- **MEDIUM (中)**: 明显异常，需要调查
- **HIGH (高)**: 严重威胁，立即处理
- **CRITICAL (严重)**: 紧急威胁，马上响应

### 告警处理建议

#### LOW级别告警

- 记录详细日志
- 持续监控相关活动
- 定期审查访问模式

#### MEDIUM级别告警

- 立即审查相关用户/活动
- 发送安全提醒通知
- 加强对该资源的监控

#### HIGH级别告警

- 立即暂停相关用户访问
- 通知安全团队紧急响应
- 启动事件响应流程

#### CRITICAL级别告警

- 立即阻断相关访问
- 启动应急预案
- 通知高级管理层
- 联系外部安全专家

## 📈 监控指标解释

### 核心安全指标

#### 系统安全评分

- **计算方式**: 基于威胁事件数量、攻击拦截率、合规状态等综合计算
- **评分范围**: 0-100分
- **更新频率**: 实时计算，每分钟刷新

#### 威胁检测率

- **定义**: 成功检测的威胁占总威胁的比例
- **目标值**: ≥ 95%
- **监控意义**: 衡量系统检测能力的有效性

#### 误报率

- **定义**: 错误识别为威胁的正常活动比例
- **目标值**: ≤ 5%
- **优化方向**: 持续调整检测算法参数

#### 平均响应时间

- **定义**: 从威胁发生到系统响应的平均时间
- **目标值**: < 100毫秒
- **重要性**: 影响威胁遏制效果

## ❓ 常见问题解答

### Q: 为什么有时会出现误报？

A: 安全系统为了保证检测覆盖率，可能会将一些正常的异常行为识别为威胁。可以通过调整检测灵敏度或添加白名单来减少误报。

### Q: 如何提高检测准确率？

A: 可以通过以下方式优化：

- 增加训练数据，完善用户行为基线
- 调整检测算法参数
- 定期更新威胁情报
- 结合多维度特征进行综合判断

### Q: 系统对性能有什么影响？

A: 系统采用轻量级设计，正常情况下对应用性能影响小于5%。可以通过调整采样率来平衡安全性和性能。

### Q: 如何处理大量告警信息？

A: 建议：

- 设置合理的告警级别过滤
- 使用告警聚合和去重功能
- 建立告警分级响应机制
- 定期审查和优化告警规则
