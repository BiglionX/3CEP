## 一、技能分类体系

### 1.1 一级分类（主类别）

| 代码    | 名称   | 英文名称      | 描述                   | 示例技能                   |
| ------- | ------ | ------------- | ---------------------- | -------------------------- |
| `DIAG`  | 诊断类 | Diagnosis     | 设备故障诊断、问题分析 | `procyc-fault-diagnosis`   |
| `ESTM`  | 估价类 | Estimation    | 设备估价、维修报价     | `procyc-estimate-value`    |
| `LOCA`  | 定位类 | Location      | 店铺查询、位置服务     | `procyc-find-shop`         |
| `PART`  | 配件类 | Parts         | 配件查询、兼容性检查   | `procyc-part-lookup`       |
| `DATA`  | 数据类 | Data          | 数据分析、报表生成     | `procyc-sales-analytics`   |
| `COMM`  | 通信类 | Communication | 通知发送、消息推送     | `procyc-send-notification` |
| `AUTO`  | 自动化 | Automation    | 流程自动化、任务编排   | `procyc-auto-reorder`      |
| `INTEG` | 集成类 | Integration   | 第三方系统集成         | `procyc-sync-inventory`    |

## 三、使用示例

### 3.1 Skill 标签配置示例

```yaml
# SKILL.md 中的 tags 配置
tags:
  # 一级分类（必填，只能选一个）
  - category:DIAG

  # 二级分类（必填，只能选一个）
  - subcategory:DIAG.HW

  # 技术栈（至少一个）
  - typescript
  - nodejs

  # 功能特性（可选）
  - ai-powered
  - real-time
  - cache-enabled

  # 集成（可选）
  - supabase

  # 质量认证（可选）
  - tested
  - documented

  # 业务场景（可选）
  - repair-shop
```

### 3.2 搜索过滤示例

**按分类搜索**:

```
GET /api/skills?category=DIAG&subcategory=DIAG.HW
```

**按标签过滤**:

```
GET /api/skills?tags=ai-powered,typescript,certified
```

**组合搜索**:

```
GET /api/skills?category=PART&tags=real-time,cache-enabled&certification=verified
```

## 四、分类与标签管理规范

### 4.1 新增分类流程

1. 提交 RFC（Request for Comments）到 `procyc-skills/rfcs`
2. 说明新增分类的必要性和使用场景
3. ProCyc 技术委员会审核（7 个工作日）
4. 审核通过后更新本文档
5. 版本号递增

### 4.2 标签使用原则

- **准确性**: 标签必须准确反映技能特性
- **最小化**: 只标注最相关的标签（5-10 个为宜）
- **一致性**: 同类技能使用相同标签集
- **可发现性**: 考虑用户搜索习惯选择标签

### 4.3 deprecated 机制

当分类或标签不再使用时：

1. 标记为 `deprecated`
2. 提供迁移指南
3. 保留至少 6 个月过渡期
4. 在下一个 MAJOR 版本移除
