# 管理模块部署和使用指南

## 📋 模块概览

本次新增了三个核心管理模块，均已通过功能回测验证：

### 1. 用户管理模块 (`/admin/user-manager`)

**核心功能**：

- 用户账户信息管理（增删改查）
- 角色权限分配（管理员、经理、员工、查看者）
- 用户状态监控（活跃、非活跃、已禁用）
- 部门组织架构管理
- 登录历史追踪
- 批量操作支持

**权限标识**：`usermgr.view`, `usermgr.manage`, `usermgr.delete`

### 2. 设备管理模块 (`/admin/device-manager`)

**核心功能**：

- 设备信息档案管理
- 在线状态实时监控
- 电池和存储状态显示
- 维修记录跟踪
- 所属关系管理（用户/店铺）
- 设备生命周期管理

**权限标识**：`devicemgr.view`, `devicemgr.manage`, `devicemgr.delete`

### 3. 系统概览模块 (`/admin/system-dashboard`)

**核心功能**：

- 核心指标仪表板（用户数、设备数、订单数等）
- 服务状态监控面板
- 用户增长趋势图表
- 设备状态分布饼图
- 系统健康度评估
- 最近活动日志

**权限标识**：`sysdash.view`, `sysdash.manage`

## 🚀 部署步骤

### 1. 文件结构确认

```
src/app/admin/
├── user-manager/           # 用户管理模块
│   └── page.tsx
├── device-manager/         # 设备管理模块
│   └── page.tsx
└── system-dashboard/       # 系统概览模块
    └── page.tsx
```

### 2. 权限配置更新

在RBAC权限系统中添加以下权限标识：

```json
{
  "usermgr.view": "查看用户管理",
  "usermgr.manage": "管理用户",
  "usermgr.delete": "删除用户",
  "devicemgr.view": "查看设备管理",
  "devicemgr.manage": "管理设备",
  "devicemgr.delete": "删除设备",
  "sysdash.view": "查看系统概览",
  "sysdash.manage": "管理系统概览"
}
```

### 3. 菜单导航集成

在管理后台菜单中添加新模块入口：

```typescript
// 示例菜单配置
const adminMenu = [
  {
    title: '用户管理',
    path: '/admin/user-manager',
    icon: 'Users',
    permission: 'usermgr.view',
  },
  {
    title: '设备管理',
    path: '/admin/device-manager',
    icon: 'Smartphone',
    permission: 'devicemgr.view',
  },
  {
    title: '系统概览',
    path: '/admin/system-dashboard',
    icon: 'Activity',
    permission: 'sysdash.view',
  },
];
```

## 📊 使用说明

### 用户管理模块使用流程

1. 访问 `/admin/user-manager`
2. 使用筛选条件快速定位用户
3. 点击"查看详情"查看完整信息
4. 点击"编辑"修改用户信息
5. 点击"删除"移除用户（需确认）

### 设备管理模块使用流程

1. 访问 `/admin/device-manager`
2. 按分类、状态、品牌等条件筛选
3. 查看设备详细信息和状态
4. 管理设备归属关系
5. 监控设备健康状况

### 系统概览模块使用流程

1. 访问 `/admin/system-dashboard`
2. 查看核心指标卡片
3. 分析趋势图表数据
4. 监控服务运行状态
5. 查看系统活动日志

## ⚠️ 注意事项

### 权限要求

- 所有模块都需要相应的查看权限才能访问
- 管理和删除操作需要额外的权限
- 建议为不同角色分配合适的权限组合

### 数据安全

- 所有敏感操作都有二次确认
- 删除操作不可逆，请谨慎操作
- 建议定期备份重要数据

### 性能优化

- 大量数据时建议使用分页加载
- 图表数据建议按需加载
- 实时监控建议设置合理的刷新间隔

## 🆘 故障排除

### 常见问题及解决方案

**Q: 页面无法访问**
A: 检查用户是否具有相应权限，确认路由配置正确

**Q: 数据加载失败**
A: 检查网络连接，确认API服务正常运行

**Q: 权限控制不生效**
A: 确认RBAC权限配置正确，清除浏览器缓存后重试

**Q: UI显示异常**
A: 检查项目UI组件库版本，确认依赖包安装完整
