# 数据库迁移 PR 审查清单

## 📋 概述

本文档为数据库迁移 Pull Request 提供标准化的审查清单，确保所有迁移都符合质量和安全标准。

## 🎯 审查目标

- 确保迁移脚本质量和一致性
- 验证回滚能力和安全性
- 确认性能影响评估
- 保证文档完整性和准确性

## ✅ 必须检查项

### 1. 基本规范检查

- [ ] **命名规范合规**
  - 正向迁移：`V{version}__{description}.sql`
  - 回滚脚本：`U{version}__{description}.sql`
  - 版本号格式正确（语义化版本）

- [ ] **文件结构完整**
  - 迁移文件存在于 `supabase/migrations/` 目录
  - 对应回滚脚本已创建
  - 文件权限设置正确

### 2. 技术质量检查

- [ ] **幂等性保障**
  - 使用 `IF NOT EXISTS` 语句
  - CREATE TABLE/INDEX 操作幂等
  - ALTER TABLE 操作安全

- [ ] **SQL 语法正确**
  - 语句语法无错误
  - 关键字大小写一致
  - 适当使用注释

- [ ] **约束和索引**
  - 必要的索引已创建
  - 外键约束合理设置
  - 唯一性约束适当

### 3. 安全性检查

- [ ] **RLS 策略配置**
  - 新表启用 Row Level Security
  - 配置适当的访问策略
  - 策略逻辑正确无安全隐患

- [ ] **数据保护**
  - 敏感数据处理得当
  - 无硬编码的敏感信息
  - 符合数据隐私要求

### 4. 回滚能力检查

- [ ] **回滚脚本存在**
  - 对应版本的回滚脚本已创建
  - 回滚脚本命名正确
  - 回滚逻辑完整

- [ ] **回滚安全性**
  - 回滚操作不会造成数据丢失
  - 破坏性操作有充分说明
  - 回滚测试已执行

## 📊 推荐检查项

### 1. 性能影响评估

- [ ] **性能考量**
  - 大表操作有分批处理考虑
  - 索引创建时机合理
  - 查询性能影响评估

- [ ] **执行时间预估**
  - 迁移执行时间合理
  - 生产环境执行窗口合适
  - 并发影响评估

### 2. 兼容性检查

- [ ] **向后兼容**
  - 不破坏现有功能
  - API 接口兼容性确认
  - 数据格式兼容性

- [ ] **环境适配**
  - 开发/测试/生产环境适配
  - 配置参数环境化
  - 依赖版本兼容性

### 3. 文档完整性

- [ ] **变更说明**
  - 清晰的变更描述
  - 影响范围说明
  - 迁移步骤详细

- [ ] **测试验证**
  - 本地测试已执行
  - 测试用例覆盖关键场景
  - 验证结果记录

## 🛠️ 审查流程

### 1. 提交前自查

开发者在提交 PR 前应完成：

```bash
# 1. 验证迁移脚本
npm run db:validate

# 2. 检查状态
npm run db:status

# 3. 本地测试执行
npm run db:migrate -- --dry-run

# 4. 填写审查清单
# 在 PR 描述中勾选相应的检查项
```

### 2. 团队审查

审查人员应按以下顺序进行：

1. **初步检查**
   - 确认基本规范符合
   - 检查文件完整性和命名
   - 验证回滚脚本存在

2. **深入审查**
   - 详细检查 SQL 逻辑
   - 评估安全性和性能影响
   - 确认测试覆盖

3. **最终确认**
   - 运行自动化检查
   - 确认所有必需项通过
   - 批准或要求修改

### 3. 合并要求

PR 只有在满足以下条件时才能合并：

- [ ] 所有必需检查项通过 ✓
- [ ] 自动化测试通过 ✓
- [ ] 至少一名资深开发者批准 ✓
- [ ] 回归测试完成 ✓

## ⚠️ 高风险变更标识

以下类型的变更多需要特别关注：

### 红色警告（禁止合并）

- [ ] 删除生产数据的迁移
- [ ] 破坏现有 API 的表结构变更
- [ ] 无回滚方案的重大重构
- [ ] 未经充分测试的复杂迁移

### 黄色警告（需要额外审查）

- [ ] 影响核心业务流程的变更
- [ ] 大表结构修改
- [ ] 复杂的数据迁移
- [ ] 性能敏感的索引变更

## 🎯 特殊场景处理

### 紧急修复

对于紧急 bug 修复：

- 优先级审查流程
- 简化文档要求
- 快速回滚预案

### 大型重构

对于大规模重构：

- 分阶段实施计划
- 详细的影响评估
- 完整的测试策略

### 数据迁移

对于复杂数据迁移：

- 数据验证机制
- 分批处理策略
- 完整的回滚计划
