## 字段说明

### 基础信息

- **名称**: 智能体的显示名称
- **领域**: 智能体所属的业务领域
- **模式**: 运行模式 (n8n 工作流 | service 服务)
- **延迟敏感性**: 对响应时间的要求 (高/中/低)
- **状态复杂度**: 状态管理和持久化复杂度 (高/中/低)
- **安全等级**: 数据安全和访问控制要求 (高/中/低)
- **流量级别**: 预期请求量级别 (高/中/低)
- **负责人**: 主要维护责任人
- **状态**: 当前部署状态

### 领域分类

- **设备识别**: 设备扫码、信息解析相关
- **用户体验**: 用户交互、推荐、引导相关
- **交易处理**: 支付、订单、财务相关
- **供应链**: 采购、供应商、议价相关
- **售后服务**: 维修、诊断、客服相关
- **数据分析**: 预测、统计、挖掘相关
- **仓储管理**: 库存、仓库、物流相关
- **设备管理**: 设备档案、生命周期、状态追踪
- **系统集成**: 智能体协调、工作流编排、服务治理

### 模式说明

- **n8n**: 基于 n8n 工作流平台的可视化智能体
- **service**: 基于代码服务的程序化智能体

### 关键属性定义

#### 延迟敏感性

- **高**: 要求毫秒级响应 (<100ms)
- **中**: 要求秒级响应 (<1s)
- **低**: 可接受较长时间响应 (>1s)

#### 状态复杂度

- **高**: 复杂的状态机，多步骤事务
- **中**: 中等复杂度状态管理
- **低**: 无状态或简单状态

#### 安全等级

- **高**: 处理敏感数据，严格访问控制
- **中**: 一般业务数据，基本安全措施
- **低**: 公开信息，较低安全要求

#### 流量级别

- **高**: 高频调用，需要水平扩展
- **中**: 中等频率，适度负载
- **低**: 低频调用，轻量负载

## 新增AI智能体详情

### 智能估值决策引擎

**功能描述**: 基于置信度的动态策略选择系统，融合ML模型、市场加权和规则引擎

**关键技术**:

- 多策略并行估值
- 置信度评估算法
- 动态权重调整
- 决策过程透明化

**接口规范**:

```
POST /api/fusion-engine/evaluate
{
  "deviceId": "device_xxx",
  "features": {...},
  "strategyWeights": {"ml": 0.7, "market": 0.2, "rule": 0.1}
}
```

### 设备生命周期管理服务

**功能描述**: 设备全生命周期事件追踪和档案管理服务

**核心能力**:

- 生命周期事件记录
- 设备状态自动推断
- 保修期智能计算
- 历史数据分析

**数据模型**:

```
DeviceLifecycleEvent {
  eventType: 'manufacture' | 'activation' | 'repair' | 'replacement' | 'recycle',
  timestamp: ISODate,
  metadata: JSON,
  triggeredBy: 'system' | 'manual' | 'integration'
}
```

### ML 置信度评估服务

**功能描述**: 多维度机器学习模型置信度计算和评估

**评估因子**:

- 数据完整性评分
- 特征质量评估
- 模型性能指标
- 市场一致性检验
- 历史准确性验证

**输出格式**:

```
{
  "confidenceScore": 0.85,
  "confidenceLevel": "high",
  "assessmentFactors": {...},
  "recommendations": [...]
}
```

## 维护说明

### 更新频率

- 核心智能体信息：每次发布更新
- 状态字段：实时更新
- 负责人信息：人员变动时更新
- AI模型版本：每月评估更新

### 管理责任

- 技术负责人：维护技术规格和状态
- 产品经理：维护业务需求和优先级
- 运维团队：维护部署状态和监控信息
- AI工程师：负责模型训练和优化

### 扩展原则

新增智能体应遵循以下原则：

1. 明确业务价值和使用场景
2. 符合现有架构和技术栈
3. 具备可测量的性能指标
4. 有清晰的责任归属
5. 支持监控和可观测性
6. 具备容错和降级机制
