# ProdCycleAI 版本更新日志

## 📋 版本管理策略

FixCycle 采用语义化版本控制 (SemVer)，版本号格式为 `MAJOR.MINOR.PATCH`：

- **MAJOR**: 不兼容的重大更新
- **MINOR**: 向后兼容的功能新增
- **PATCH**: 向后兼容的问题修正

## 📝 升级说明

### 从 v2.x 升级到 v3.0

#### 数据库迁移
```bash
# 执行数据库迁移脚本
npm run db:migrate

# 验证迁移结果
npm run db:validate
```

#### 配置文件更新
```bash
# 更新环境变量
cp .env.example .env.production
# 编辑新的配置项

# 重启服务
pm2 restart fixcycle-app
```

#### 前端资源更新
```bash
# 构建新版本
npm run build

# 清除CDN缓存
# 在CDN控制台执行缓存刷新
```

### 从 v1.x 升级到 v2.0

#### 重要变更
- 数据库结构重大调整
- API接口路径变更
- 认证机制升级

#### 迁移步骤
1. 备份现有数据
2. 执行渐进式迁移
3. 验证功能完整性
4. 切换到新版本
