## 🛠️ 技术架构亮点

### 1. API 设计规范

- **统一响应格式**: `{code, message, data, timestamp}`
- **标准化错误码**: 400xx(参数错误), 401xx(认证错误), 500xx(服务器错误)
- **版本化管理**: `/api/v1/` 路径前缀
- **RESTful 设计**: 符合 HTTP 语义的资源操作

### 2. 缓存系统

- **双层缓存架构**: Redis + 内存缓存后备
- **智能降级**: Redis 不可用时自动切换到内存缓存
- **缓存策略**: 热点数据 1 小时缓存，用户数据 5 分钟缓存
- **缓存穿透防护**: 空值缓存和布隆过滤器

### 3. 安全机制

- **JWT 认证**: Bearer Token 方式
- **权限控制**: 基于角色的访问控制(RBAC)
- **数据隔离**: 用户数据访问权限控制
- **输入验证**: 严格的参数校验和 SQL 注入防护

### 4. 性能优化

- **数据库优化**: 索引优化和查询优化
- **批量处理**: 支持批量操作减少网络开销
- **异步处理**: 非阻塞 IO 操作
- **连接池**: 数据库连接复用
