## 产品需求

在现有企业溯源系统基础上，集成区块链私有链技术，实现产品防伪溯源功能。

## 核心功能需求

1. **区块链智能合约**：编写并部署 Solidity 智能合约，用于存储产品防伪码和溯源记录
2. **链上数据存储**：将产品防伪码、溯源关键节点数据上链存储，确保不可篡改
3. **防伪验证机制**：消费者扫码验证产品真伪，查询区块链上的防伪信息
4. **不可篡改的溯源记录**：覆盖生产、流通、售后等全链路溯源记录上链

## 业务场景

- 批量生成防伪码（绑定产品）
- 扫码验证真伪（消费者端）
- 溯源记录上链（生产/物流/售后等环节）

## 技术约束

- 区块链平台：私有链（自建）
- 系统关系：增强模式（叠加在现有数据库溯源之上）
- 现有溯源模块：`/enterprise/admin/traceability`（已实现）
- 数据库表结构：`docs/technical-docs/enterprise-traceability-tables.sql`

## 技术方案

### 区块链选型

- **私有链方案**：Hyperledger Besu（企业级 EVM 兼容）或 Geth Clique PoA 联盟链
- **开发/测试环境**：Ganache（本地模拟私有链）
- **智能合约语言**：Solidity ^0.8.0
- **Web3 交互库**：ethers.js v6 或 viem

### 模块设计

#### 1. 智能合约层 (contracts/)

- `ProductAuth.sol`：核心防伪溯源合约
- `registerProduct()`: 注册产品防伪码
- `verifyProduct()`: 验证产品真伪
- `addTraceabilityRecord()`: 添加溯源记录
- `getProductInfo()`: 查询产品信息
- `getTraceabilityHistory()`: 查询溯源历史

#### 2. 区块链服务层 (src/lib/blockchain/)

- `contracts/`: 编译后的 ABI 和字节码
- `ProductContract.ts`: 智能合约交互封装
- `BlockchainService.ts`: 区块链操作服务
- `config.ts`: 私有链节点配置

#### 3. API 扩展层 (src/app/api/enterprise/traceability/)

- 新增区块链相关 API 端点
- 与现有 API 协同工作

### 实现策略

- 采用「增强模式」：区块链作为可信存证层，数据库保留完整业务数据
- 智能合约设计支持批量操作，降低 Gas 成本
- 链上仅存储关键哈希值，原始数据仍在数据库
- 设计回调机制处理区块链交易确认

### 性能考虑

- 批量上链采用队列机制，避免阻塞主线程
- 链上数据查询结果缓存
- 异步处理区块链交易确认

## 设计说明

本计划主要涉及后端区块链集成和现有前端页面的功能增强，不涉及全新 UI 设计。

现有溯源管理页面 (`/enterprise/admin/traceability`) 需增加以下功能入口：

- 区块链同步状态显示
- 上链操作按钮
- 链上验证入口

消费者验证页面可复用现有二维码扫描模块，增加区块链验证结果显示。
