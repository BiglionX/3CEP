## 技术架构特点

### 🤖 智能混合策略
```
混合估值 = ML预测 × 0.7 + 规则估值 × 0.3
```
- **优势**: 结合机器学习的精准性和规则引擎的稳定性
- **可靠性**: 双重保障，任一系统故障不影响服务

### 🔄 完善的回退机制
- ML服务不可用 → 自动切换规则引擎
- 特征提取失败 → 使用默认特征值
- 模型预测异常 → 回退到纯规则估值

### 📊 多维度特征体系
**硬件特征**: 年龄、RAM、存储、CPU性能评分
**使用特征**: 维修次数、更换部件数、转让历史
**成色特征**: 屏幕、电池、机身、功能完整度
**衍生特征**: 年龄-RAM比率、维修频率等

### ⚡ 高性能设计
- 单次估值平均耗时 < 200ms
- 支持批量预测处理
- 内存友好的特征处理
- 异步非阻塞架构

## 部署架构

### 服务组件
```
估值API服务 (3001端口)
    ↓
估值引擎服务
    ├── 规则引擎 (始终可用)
    └── ML客户端
        ├── 本地简化模型 (开发环境)
        └── 远程ML服务 (生产环境 3002端口)
```

### 数据流向
```
设备档案 + 成色信息
    ↓
特征提取器
    ↓
混合估值策略
    ├── ML模型预测
    └── 规则引擎估值
    ↓
加权平均结果
    ↓
返回最终估值
```
