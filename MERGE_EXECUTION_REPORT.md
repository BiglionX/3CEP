## 📝 关键决策说明

### 为什么选择强制重置而非合并？

1. **线性历史关系**: master 已经包含 250313 的全部提交，不是分叉关系
2. **避免冲突**: 递归合并会产生不必要的合并提交和潜在冲突
3. **清理错误**: master 有大量未提交的删除操作（可能是误操作）
4. **简化历史**: 保持提交历史清晰易懂
5. **符合目标**: 你的目标是"保留最全面正确代码"，250313 更稳定

### 风险评估

| 风险项       | 等级 | 缓解措施                   |
| ------------ | ---- | -------------------------- |
| 丢失新功能   | 低   | 已创建备份分支，可随时恢复 |
| 团队协作冲突 | 中   | 需通知团队成员即将强制推送 |
| 远程分支保护 | 低   | 如有保护规则需先解除       |
| 数据丢失     | 极低 | 多重备份（分支 + stash）   |
