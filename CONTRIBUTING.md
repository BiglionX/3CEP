## 🚀 快速开始

### 1. Fork 仓库

```bash
# Fork 项目到您的 GitHub 账户
# Clone 到本地
git clone https://github.com/your-username/3cep.git
cd 3cep
```

### 2. 环境设置

```bash
# 安装依赖
npm install

# 设置开发环境
npm run setup:env

# 启动开发服务器
npm run dev
```

### 3. 创建分支

```bash
git checkout -b feature/your-feature-name
# 或
git checkout -b fix/your-bug-fix
```

## 📝 开发规范

### 代码风格

- 遵循项目现有的 ESLint 和 Prettier 配置
- 使用 TypeScript 进行类型检查
- 函数和组件需要适当的注释

### 提交信息规范

```bash
# 格式：type(scope): description
feat(auth): 添加用户认证功能
fix(api): 修复数据查询bug
docs(readme): 更新安装说明
style(ui): 调整按钮样式
refactor(utils): 重构工具函数
test(login): 添加登录测试用例
```

### Pull Request 要求

- PR 标题清晰描述变更内容
- 包含相关 Issue 编号（如果有）
- 通过所有自动化测试
- 代码审查后合并

## 🧪 测试

### 运行测试

```bash
# 单元测试
npm run test

# 端到端测试
npm run test:e2e

# 代码覆盖率
npm run test:coverage
```

### 测试要求

- 新功能必须包含测试用例
- Bug 修复需要添加回归测试
- 测试覆盖率不低于 85%
