## 🔧 技术难点与解决方案

### 难点 1: GitHub API 速率限制

**问题**: 未认证情况下每小时仅 60 次请求

**解决方案**:

1. 实现智能缓存（TTL 5 分钟）
2. 支持 Token 认证（提升至 5000 次/小时）
3. 批量请求优化（减少调用次数）
4. 降级方案（使用过期缓存）

**效果**: 实际 API 调用减少 80%

### 难点 2: SSR 与客户端数据同步

**问题**: 服务端渲染时无法访问客户端缓存

**解决方案**:

1. 创建 `useGitHubRepoFromCache()` Hook
2. 仅在客户端尝试从缓存读取
3. 服务端跳过 API 调用
4. 水合后自动同步数据

**效果**: 无闪烁、无重复请求

### 难点 3: 缓存一致性

**问题**: 多个组件同时请求同一仓库数据

**解决方案**:

1. 单例 Map 存储全局缓存
2. 引用透明性保证
3. 统一的 TTL 策略
4. 集中式缓存管理

**效果**: 所有组件数据一致
