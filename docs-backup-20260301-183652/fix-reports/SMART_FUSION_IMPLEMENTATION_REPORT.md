## 系统架构设计

### 核心组件关系

```
┌─────────────────┐    ┌────────────────────┐    ┌─────────────────┐
│   用户请求层    │───▶│  智能决策引擎V2    │───▶│   策略执行层    │
└─────────────────┘    └────────────────────┘    └─────────────────┘
                              │                           │
                              ▼                           ▼
                    ┌─────────────────┐    ┌─────────────────┐
                    │   置信度评估    │    │  ML/市场/规则   │
                    └─────────────────┘    └─────────────────┘
                              │
                              ▼
                    ┌─────────────────┐
                    │   决策日志记录  │
                    └─────────────────┘
```

### 数据流向

1. **输入处理**: 接收设备信息、成色状态、市场价格等参数
2. **并行估值**: 同时调用ML模型、市场加权、规则引擎三种策略
3. **置信度评估**: 分别评估各策略的可靠性
4. **策略选择**: 根据预设规则和实时置信度动态选择最优策略
5. **结果返回**: 提供最终估值、使用策略、置信度等完整信息
6. **反馈收集**: 记录用户反馈用于后续模型优化

## 技术亮点

### 1. 智能策略融合

- **动态权重调整**: 根据数据质量实时调整各策略权重
- **多维度评估**: 综合考虑数据完整性、时效性、一致性等因素
- **优雅降级**: 当高优先级策略不可用时自动切换到备选方案

### 2. 置信度驱动决策

- **分层阈值**: 高/中/低三个置信度等级对应不同的策略选择逻辑
- **量化评估**: 将主观判断转化为可量化的数值评估
- **透明解释**: 清晰说明决策依据和选择理由

### 3. 系统可扩展性

- **插件化设计**: 易于添加新的估值策略
- **配置驱动**: 通过配置文件调整系统行为
- **模块化解耦**: 各组件职责清晰，便于维护和升级
