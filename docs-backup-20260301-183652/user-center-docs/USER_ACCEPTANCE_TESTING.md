# 📱 扫码落地页设备档案扩展 - 用户验收测试指南

## 🎯 测试概述

本测试指南将帮助您验证扫码落地页设备档案扩展的所有功能是否正常工作。

## 🔧 前置条件

1. **开发服务器运行**: 确保 Next.js 开发服务器正在运行在 `http://localhost:3001`
2. **浏览器准备**: 准备现代浏览器（Chrome、Firefox、Safari 等）

## 🧪 功能测试清单

### 1. 基础页面访问测试 ✅

**测试步骤**:

1. 打开浏览器访问: `http://localhost:3001/scan/test_device_001`
2. 观察页面是否正常加载

**预期结果**:

- ✅ 页面正常显示，无错误信息
- ✅ 显示设备基本信息
- ✅ 页面布局完整

### 2. Tab 切换功能测试 ✅

**测试步骤**:

1. 访问扫码落地页
2. 点击"说明书/AI 诊断"标签页
3. 点击"设备档案"标签页
4. 观察两个标签页之间的切换效果

**预期结果**:

- ✅ Tab 切换流畅，有平滑过渡动画
- ✅ 点击后正确激活对应标签页
- ✅ 内容区域正确显示对应内容
- ✅ 移动端触控操作正常

### 3. 设备状态标识测试 ✅

**测试步骤**:

1. 访问扫码落地页
2. 查看页面顶部的设备状态显示

**预期结果**:

- ✅ 显示设备当前状态（如：正常使用中）
- ✅ 状态徽章颜色符合设计规范
- ✅ 状态文本清晰易读

### 4. 设备档案 Tab 功能测试 ✅

**测试步骤**:

1. 切换到"设备档案"标签页
2. 观察档案摘要信息显示
3. 查看事件列表展示
4. 检查事件排序（按时间倒序）

**预期结果**:

- ✅ 显示设备档案摘要信息
- ✅ 事件列表按时间倒序排列
- ✅ 每个事件都有对应的图标和描述
- ✅ 事件时间格式正确显示

### 5. EventCard 组件测试 ✅

**测试步骤**:

1. 在设备档案 Tab 中查看各个事件卡片
2. 特别关注维修事件卡片
3. 点击维修事件的"查看详情"按钮

**预期结果**:

- ✅ 不同类型事件显示不同的图标
- ✅ 事件时间、地点、备注信息完整
- ✅ 维修事件可点击查看详细信息
- ✅ 卡片样式美观，响应式适配良好

### 6. 响应式设计测试 ✅

**测试步骤**:

1. 在桌面浏览器中测试
2. 调整浏览器窗口大小
3. 在移动设备上测试（或使用浏览器开发者工具模拟移动设备）

**预期结果**:

- ✅ 在不同屏幕尺寸下布局合理
- ✅ Tab 切换在移动端操作顺畅
- ✅ 文字大小适中，易于阅读
- ✅ 图标和按钮大小适合触控操作

## 🔍 故障排除

如果某些测试失败，请检查：

1. **开发服务器状态**: 确保 `npm run dev` 正在运行
2. **端口冲突**: 确认使用的是 3001 端口
3. **文件完整性**: 确认所有必需文件都存在且无语法错误
4. **网络连接**: 确保本地网络连接正常
