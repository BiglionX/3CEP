### 2. 错误处理策略

#### 错误分类逻辑

```typescript
// 网络错误识别
if (
  message.includes('network') ||
  message.includes('fetch') ||
  message.includes('timeout')
) {
  type = NETWORK_ERROR;
  severity = HIGH;
  userMessage = '网络连接不稳定，请检查网络后重试';
}

// 服务器错误识别
if (message.includes('server') || message.includes('500')) {
  type = SERVER_ERROR;
  severity = CRITICAL;
  userMessage = '服务器暂时不可用，请稍后再试';
}
```

#### 重试机制配置

```typescript
const config = {
  maxRetries: 2, // 最大重试次数
  retryDelay: 1500, // 重试延迟(ms)
  autoRetry: true, // 是否自动重试
  showUserNotifications: true, // 显示用户通知
  logToConsole: true, // 控制台日志
};
```

## ⚠️ 风险识别与控制

### 已识别风险

1. **过度通知风险**: 频繁的错误提示可能打扰用户
   - **控制措施**: 实现错误频率限制和重要性分级

2. **隐私泄露风险**: 错误报告可能包含敏感信息
   - **控制措施**: 敏感信息脱敏处理，生产环境限制详细错误信息

3. **性能影响风险**: 错误处理逻辑可能影响应用性能
   - **控制措施**: 异步错误处理，最小化主线程阻塞
