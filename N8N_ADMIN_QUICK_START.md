# n8n Admin 管理页面快速启动指南

## 🚀 立即开始使用

### 1️⃣ 配置环境变量

在 `.env` 文件中添加以下配置：

```bash
# n8n API 配置
NEXT_PUBLIC_N8N_URL=http://localhost:5678
N8N_API_TOKEN=your-n8n-api-token-here
```

**获取 N8N_API_TOKEN**:

1. 登录 n8n 管理后台
2. 进入 Settings → API
3. 复制 API Token

### 2️⃣ 启动开发服务器

```bash
npm run dev
```

### 3️⃣ 访问管理页面

浏览器访问：`http://localhost:3001/admin/agents`

## 🎯 快速操作指南

### 添加第一个凭证

1. 访问 `/admin/agents/credentials`
2. 点击 "添加凭证" 按钮
3. 选择凭证类型（如 API Key）
4. 填写名称、提供商和密钥值
5. 点击 "创建凭证"

### 导入工作流模板

1. 访问 `/admin/agents/templates`
2. 浏览或搜索需要的模板
3. 点击 "使用此模板" 按钮
4. 或点击 "导入模板" 从 URL 导入

### 创建定时任务

1. 访问 `/admin/agents/schedules`
2. 点击 "创建调度"
3. 填写任务名称和 Cron 表达式
4. 选择关联的工作流
5. 设置时区并保存

### 邀请团队成员

1. 访问 `/admin/agents/team`
2. 切换到 "邀请列表" Tab
3. 点击 "邀请成员" 按钮
4. 输入邮箱和选择角色
5. 发送邀请链接

## 🔧 常见问题

### Q: 页面显示 "加载中..." 但不加载数据？

**A**: 检查以下几点：

1. n8n 服务是否已启动（默认端口 5678）
2. `NEXT_PUBLIC_N8N_URL` 配置是否正确
3. `N8N_API_TOKEN` 是否有效
4. 浏览器控制台是否有错误信息

### Q: 如何查看真实的 n8n 数据？

**A**:

1. 确保本地 n8n 实例正在运行
2. 配置正确的 API Token
3. 页面会自动连接真实 API
4. 如果连接失败，会显示模拟数据用于演示

### Q: 凭证安全吗？

**A**:

- ✅ 所有敏感数据都会加密存储
- ✅ 密钥值默认隐藏显示
- ✅ 支持一键复制
- ✅ 符合 n8n 安全标准

### Q: 可以自定义这些页面吗？

**A**:
当然可以！所有页面都是开源的 TypeScript/React 代码：

- 位置：`src/app/admin/agents/*/page.tsx`
- 使用统一的 UI 组件库
- 易于修改和扩展

## 📖 Cron 表达式示例

| 表达式        | 含义            | 使用场景     |
| ------------- | --------------- | ------------ |
| `0 * * * *`   | 每小时执行      | 定期检查任务 |
| `0 2 * * *`   | 每天凌晨 2 点   | 每日备份     |
| `*/5 * * * *` | 每 5 分钟       | 高频监控     |
| `0 9 * * 1`   | 每周一上午 9 点 | 周报生成     |
| `0 0 1 * *`   | 每月 1 号零点   | 月度结算     |
